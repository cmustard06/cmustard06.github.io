序列化与反序列化

实验内容

基本概念

把数据结构或者对象转换为字节序列的过程称为对象的序列化

把字节序列恢复为对象或者数据结构的过程称为对象的反序列化

​ 序列化主要有两种用途

​ 1)、将对象或者数据结构对应的字节序列直接储存在文件或者数据库中，以方便其他程序使用

​ 2)、方便将对象或者数据结构用于网络传输，限于同一种编程语言

以下文档将主要以PHP的序列化为例进行说明

漏洞形成原理

PHP 将所有以 __（两个下划线）开头的类方法保留为魔术方法 ，比如 __construct, __destruct, __toString,__sleep,__wakeup等等。

__sleep()和 __wakeup():serialize() 函数会检查类中是否存在一个魔术方法__sleep()。如果存在，该方法会先被调用，然后才执行序列化操作。此功能可以用于清理对象，并返回一个包含对象中所有应被序列化的变量名称的数组。如果该方法未返回任何内容，则 NULL 被序列化 。相反地，unserialize() 从字节流中创建了一个对象之后,马上检查是否具有__wakeup 的函数的存在。如果存在，__wakeup立刻被调用。使用 _wakeup 的目的是重建在序列化中可能丢失的任何数据库连接以及处理其它重新初始化的任务 。

__toString()方法用于一个类被当成字符串时应怎样回应。例如 echo $obj; 应该显示些什么。此方法必须返回一个字符串

__construct:在一个类中定义一个方法作为构造函数。具有构造函数的类会在每次创建新对象时先调用此方法

__destruct:析构函数会在到某个对象的所有引用都被删除或者当对象被显式销毁时执行。

serialize() 返回字符串，此字符串包含了表示 value 的字节流，可以存储于任何地方。这有利于存储或传递 PHP 的值，同时不丢失其类型和结构。想要将已序列化的字符串变回 PHP 的值，可使用 unserialize()。serialize()可处理除了 resource 之外的任何类型。甚至可以 serialize() 那些包含了指向其自身引用的数组。

漏洞利用

最后访问存在反序列漏洞页面

http://127.0.0.1/test.php

在正常情况下，该页面实现的是一个简单的通过get方法获取一个已经被序列化的字符串，然后在进行反序列化，并将结果返回给浏览器中的demo。请求http://127.0.0.1/test.php?usr_serialized=O:7:"Student":2:{s:6:"number";i:66;s:4:"name";s:4:"Teng";}，服务器会对usr_serialized参数提交的值进行反序列化。

反序列化成功，服务器返回到浏览器的结果如下

这是正常的访问，但是我们通过查看源代码发现，如果我们构造一个序列化字符串，在反序列化之后去调用FileClass，然后读取我们目标主机中建立的一个敏感的pass.txt文件，该文件中存放了一些敏感信息，通过审计源代码中的FileClass类

发现该类声明了一个__toString()方法，我们知道该方法在我们使用print,echo直接输出对象的时候就会被调用，因此，我们可以构造一个序列化字符串，该序列化字符串的反序列化之后的作用是调用FileClass读取pass.txt文件中的敏感内容（ps:也可以是其他文件）。

实验分析与总结

如何解决和避免它,不要对用户提供的数据使用“反序列化”。 可以使用“json_decode”。

对于反序列话漏洞即使可能很难发现，甚至很难加以利用，但这仍然是一个非常危险的漏洞。 它可能会导致拒绝服务，任意文件读取，SQL注入，远程代码执行等漏洞。

数组与数组切片

notes
  • golang
  • 基础

数组是go语言中最常见的数据结构之一，数组时有同一类型数据组成的集合

0x01 数组声明

在go语言中，数组的长度在定义后就不能够进行更改，在声明长度是可以是一个常量或者一个常量表达式，可以使用go语言内置的len()函数来获取数组的长度

元素访问

  • 可以使用数组下标来进行访问，下标都是从0开始的

遍历结果

  • range关键字用于便捷的遍历容器中的元素

range有两个返回值，一个是元素的数组下标，第二个是元素的值
结果

值类型

在go语言中，数组是一个值类型，所有的值类型变量在复制和作为参数传递时都将产生一次复制动作。简单来说就是如果将数组作为函数的参数类型，那么将数组传入到函数内部进行修改，并不会影响原数组的值。

数组切片

由于数组在定义之后，数组的长度无法修改，数组是值类型，每传递一次都会产生一份副本，因此Go语言提供了数组切片来弥补数组的不足

数组切片拥有自己的数据结构，而不单单是一个指针。数组切片的数据结构可以抽象为以下3个变量

  • 一个指向原生数组的指针
  • 数组切片中的元素个数
  • 数组切片已经分配的存储空间
    基于数组，数组切片添加了一系列的管理功能，可以随时动态扩充存放空间。

    创建数组切片

    两种方法，一种是基于数组创建，另一种是直接创建。

    基于数组创建

go语言支持使用myarray[first:last]来基于数组生成一个数组切片。
基于所有元素的数组切片
mySlice = myarray[:]
基于前五个元素的数组切片
mySlice = myarray[:5]
从第五个元素开始的所有元素创建数组切片
mySlice = myarray[5:]

直接创建

go语言提供的内置函数make()可以用于灵活的创建数组切片

元素遍历

元素遍历与数组的元素遍历一样

动态增减元素

与数组相比，数组切片多了一个存储能力的概念，即元素个数和分配的地址空间可以是两个不同的值，合理的设置存储能力，可以大幅降低数组切片内部重新分配内存额搬送内存块的频率。

数组切片支持内置的cap() 和len()函数，前者是返回数组切片分配的空间大小，后者是返回的数组切片中当前存储的元素个数

append()函数，可以在数组切片的尾部追加元素，从而生成一个新的数组切片
mySlice = append(mySlice,1,2,3,4)
append()函数的第二个参数是一个不定参数，可以按照自己的需求添加若干个元素，也可以直接将一个数组切片追加到另一个数组切片的末尾。

基于数组切片创建数组切片

内容复制

copy()函数用于将内容从一个数组切片复制到另一个数组切片，如果加入的两个数组切片不一样大，就会按照其中最小的那个数组切片的元素个数进行复制。

map

map是一对一堆键值对的未排序集合

变量声明

map声明基本没有多余的元素
var myMap map[string] int
其中，myMap是声明的map的变量名，string是键的类型，int是所存放的值的类型

##创建
可以使用内置的make()函数来创建一个新map
以下是创建一个键类型为string、值类型为自定义的PersonInfo的map:
myMap = make(map[string] PersonInfo)
也可以选择是否在创建时指定该map的初始能力，下面创建一个初始存储能力为100的map:
myMap = make(map[string] PersonInfo, 100)
创建并初始化map的代码如下：
myMap = map[string] PersonInfo{"1234":PersonInfo{"1","Jack","Room"}, }

元素赋值

myMap["1234"] = PersonInfo{"1","Jack","Room..."}

元素删除

内置delete()函数，用于删除容器内的元素。
delete(myMap, "1234")
从myMap中删除键为“1234”的键值对，如果不存在，什么都不会发生，如果传入的变量的值是nil，该调用将导致程序抛出异常
